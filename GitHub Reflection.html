My GitHub Publishing Experience

I hadn’t ever submitted to GitHub before, so this was very interesting.

I started with creating my account, which had the worst captcha I’ve ever completed. I was worried I would be labelled a robot. From there, I got the base page set up with a README allowed and left visibility private, with hope that I could share it out from there for the assignment.

I didn’t assign it a license either because it was to be private. I made a copy of my HTML version of my photo essay, then renamed it index.html before uploading. Then, I made new copies of my images with shorter names and added them as well. Previously, I hosted my images from my own website. Since I was moving them to GitHub, I had to adjust the file names within the HTML. I made the adjustments within GitHub for practice and to create a version history as well. The images were added successfully.

At this point, I wanted to confirm that it was working, but I didn’t know how to view the file from there. I opened your lecture video and followed along to publish. I realized making it private was a mistake and had to go into the “Danger Zone” settings to make it public. I went back to Pages and after a few minutes it was live!

I then wrote up a README in Markdown and added it to the README file. Before finishing up, I wanted to adjust the size of the images in my HTML, as they were rather large. I added width="50%" and recommitted, then waited for the update. It didn’t work, so I adjusted it to 400 instead to try a specific pixel size. My images looked great then!

This was a great exercise and gave me good practice in using GitHub. It’s always been a little intimidating, so using it hands on really helped me understand it better. Now I’ll add this to a new HTML, link it in the original, and finish this thing up!
